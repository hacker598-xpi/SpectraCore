SpectraCore – Documentación completa

Versión 3.0.0 – Biblioteca de análisis espectral en tiempo real con FFT acelerada por SIMD.

1. Introducción

SpectraCore es una biblioteca C++20 de alto rendimiento diseñada para transformar audio en espectros frecuenciales en tiempo real. Ideal para visualizadores musicales, ecualizadores, medidores de espectro y cualquier aplicación que necesite procesar señales de audio con mínima latencia.

Características principales:

  • FFT real optimizada con SSE4.1, AVX2, AVX‑512, NEON (despacho automático en tiempo de ejecución).
  • Ventanas seleccionables: Hann, Hamming, Blackman, Kaiser.
  • Solapamiento configurable (hop size).
  • Suavizado exponencial de magnitudes.
  • Agrupación logarítmica de frecuencias (energía por banda).
  • Salida de magnitud lineal, dB y fase por bin.
  • Doble búfer de entrada y salida para operación segura entre hilos.
  • Modo asíncrono integrado (AsyncAnalyzer) con espera pasiva.
  • Multiplataforma: Windows, macOS, Linux, iOS, Android.

2. Instalación

Compilación desde fuentes

git clone https://github.com/tu-usuario/SpectraCore.git
cd SpectraCore
./build.sh

El script build.sh lee la configuración de config.txt y genera:

  • Biblioteca estática (libspectracore.a)
  • Biblioteca dinámica (libspectracore.so o .dylib)
  • Paquete de distribución (libspectracore-3.0.0-linux-x86_64.tar.gz)

El paquete contiene los encabezados, bibliotecas y esta documentación, listo para usar en otros proyectos.

Uso en CMake

find_package(SpectraCore REQUIRED)
target_link_libraries(mi_app PRIVATE SpectraCore::spectracore)

O enlaza manualmente:

g++ -std=c++20 mi_app.cpp -I/path/SpectraCore/include -L/path/lib -lspectracore -o mi_app

3. Conceptos básicos

Analizador (Analyzer): es el corazón de la librería. Toma muestras de audio, las agrupa en cuadros, aplica ventana, calcula la FFT y entrega el espectro.

Flujo de datos:

  1. Hilo de audio: llama a pushSamples() con las muestras.
  2. Hilo de proceso: llama a processReadyFrame() para calcular el espectro.
  3. Hilo de interfaz: obtiene el resultado con getSpectrum().

El doble búfer interno garantiza que no haya bloqueos entre productor y consumidor.

4. API de Analyzer

Constructor

Analyzer();

Inicializa el analizador con ventana Hann, 50% de solapamiento y tasa de muestreo 44100 Hz.

Configuración

setWindowType(type, beta=6.0)

void setWindowType(WindowType type, float kaiserBeta = 6.0f) noexcept;

Establece la ventana aplicada a cada cuadro antes de la FFT.

  • WindowType::Hann
  • WindowType::Hamming
  • WindowType::Blackman
  • WindowType::Kaiser (usa kaiserBeta)

setOverlapFactor(factor)

void setOverlapFactor(size_t factor) noexcept;

El avance entre cuadros será hopSize = N / factor. Por ejemplo, factor=2 produce 50% de solapamiento.

setHopSize(hop)

void setHopSize(size_t hop) noexcept;

Define directamente el número de muestras a avanzar entre cuadros. Valores pequeños aumentan el solapamiento y la fluidez.

setSmoothing(alpha)

void setSmoothing(float alpha) noexcept;

Aplica suavizado exponencial a las magnitudes lineales:

smoothed = alpha * nuevo + (1 - alpha) * anterior

alpha=1.0 desactiva el suavizado; alpha=0.0 congela la salida. Típico: 0.2–0.4.

setLogBands(numBands, minFreq, maxFreq, sampleRate)

void setLogBands(size_t numBands, float minFreq, float maxFreq, float sampleRate) noexcept;

Agrupa los bins de la FFT en bandas logarítmicas. La energía (magnitud²) de cada bin se acumula en la banda correspondiente y luego se convierte a dB. numBands=0 desactiva la agrupación.

Alimentación de datos

pushSamples(samples)

void pushSamples(std::span samples) noexcept;

Envía un bloque de muestras consecutivas. La clase acumula internamente hasta completar un cuadro.

Procesamiento

hasNewFrame()

bool hasNewFrame() const noexcept;

Devuelve true si hay un cuadro listo para procesar (no consume el cuadro).

waitForFrame()

void waitForFrame() const noexcept;

Bloquea el hilo hasta que haya un nuevo cuadro disponible. Útil en modo asíncrono.

processReadyFrame()

void processReadyFrame() noexcept;

Toma el cuadro disponible, ejecuta la FFT real, aplica suavizado, bandas logarítmicas y actualiza las salidas.

Obtención del espectro

getSpectrum()

struct Spectrum {
    std::span magnitudeLin;   // N/2+1 bins
    std::span magnitudeDB;    // N/2+1 bins
    std::span phase;          // N/2+1 bins
    std::span logMagnitudes;  // numLogBands (vacío si no activado)
};

Spectrum getSpectrum() const noexcept;

Retorna vistas de solo lectura a los buffers internos. Los datos son consistentes gracias al doble búfer de salida.

5. Modo asíncrono: AsyncAnalyzer

Para integrar rápidamente sin gestionar hilos manualmente:

SpectraCore::AsyncAnalyzer<2048> async;

async.start([](const auto& spectrum) {
    // Este callback se ejecuta cada vez que hay un nuevo espectro.
    // spectrum.magnitudeDB, spectrum.phase...
});

// En el hilo de audio:
async.analyzer().pushSamples(miBuffer);

// Al finalizar:
async.stop();

El hilo interno espera pasivamente (sin consumir CPU) hasta que hay datos.

6. Ejemplos prácticos

Visualizador básico

#include <SpectraCore/Analyzer.h>

int main() {
    constexpr size_t N = 2048;
    SpectraCore::Analyzer analizador;

    analizador.setWindowType(SpectraCore::WindowType::Hann);
    analizador.setOverlapFactor(2);               // 50% solapamiento
    analizador.setSmoothing(0.3f);
    analizador.setLogBands(32, 20.0f, 20000.0f, 44100.0f);

    std::vector audio(512);
    // Llenar audio...

    analizador.pushSamples(audio);
    if (analizador.hasNewFrame()) {
        analizador.processReadyFrame();
        auto spec = analizador.getSpectrum();
        for (size_t i = 0; i < 32; ++i)
            std::cout << "Banda " << i << ": " << spec.logMagnitudes[i] << " dB\n";
    }
    return 0;
}

8. Rendimiento

Las rutinas SIMD están vectorizadas manualmente con intrínsecos y los twiddles preempaquetados. En pruebas con N=2048 y AVX2, SpectraCore iguala o supera a FFTW3 en escenarios de baja latencia. Para mediciones exactas, active BUILD_BENCHMARKS y ejecute ./benchmark.
